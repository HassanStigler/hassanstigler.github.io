API & Platform Leadership

How I turned GoFileRoom into an extensible, revenue-driving platform

At Thomson Reuters, APIs were not a technical side project — they were a strategic product lever. I owned the API and platform strategy for GoFileRoom, a $40M+ enterprise document management platform serving tax and accounting firms across North America.

The engineering teams built the services. I owned the platform vision, roadmap, governance, and commercial outcomes.

Why the API Program Existed

GoFileRoom was deeply trusted by customers, but it was locked behind a web UI. That limited automation, slowed firms down, and prevented GoFileRoom from becoming part of a larger ecosystem.

The API program had three strategic goals:

  • Allow customers to automate document workflows without logging into the UI
  • Enable internal Thomson Reuters products to integrate with GoFileRoom
  • Create a foundation for new partner and ecosystem integrations

This made GoFileRoom more than a product — it made it a platform.

What I Owned

I was not writing production code. I was running the API business.

  • API and webhook roadmap aligned to customer and revenue priorities
  • REST and SOAP strategy, versioning, and endpoint governance
  • Developer experience, documentation, and discoverability
  • Internal and external API consumers and support models
  • Usage telemetry, reliability, and commercial alignment

Developer Experience & Documentation

When I took ownership, many APIs technically existed, but they were not usable. Endpoints were listed as raw URIs with no descriptions, summaries, or guidance.

I led the migration to a modern developer portal and restructured how APIs were presented:

  • Endpoints grouped into logical collections
  • Human-readable summaries and descriptions added to every endpoint
  • Use-case driven API catalogs instead of raw technical lists

Webhooks & Workflow Automation

Beyond APIs, I led the delivery of webhook-based workflows that allowed firms to react to document events in real time — removing manual polling and human intervention.

These capabilities were used by both external customers and internal Thomson Reuters teams to build new services on top of GoFileRoom.

From Stickiness to Monetization

The APIs were initially included to drive platform stickiness and retention.

Using telemetry and dashboards I built, we could see which customers were making high volumes of API calls. That data allowed us to create a premium usage tier for customers who needed higher throughput and deeper integrations.

APIs went from a cost center to a monetizable platform capability.

The Business Result

  • Customers automated work that previously required multiple staff members
  • Internal product teams unblocked their own roadmaps through GoFileRoom APIs
  • Platform stickiness increased through deep integrations
  • API usage became a lever for premium monetization

This API program was a key part of how GoFileRoom grew from $38M to over $43M in annual revenue while rebuilding customer trust.
